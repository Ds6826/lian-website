Every fact is bitemporal, every change is admitted through a policy gate, and every event lands in a tamper-evident hash chain — so you can prove what an agent knew, when it knew it, and that the record was never altered.

Five layers, one API.

Bitemporal model

Ask what was true, not just what is stored.

Every fact carries event time (when it happened) and system time (valid_from / valid_to). Recall runs through a validity gate: present queries return facts where valid_to IS NULL; point-in-time queries return facts where valid_from ≤ as_of < valid_to.

So recall_at(...) reconstructs exactly what an agent knew on any past date — the foundation for backtests, snapshots, and lineage.

Recall pipeline

Hybrid search, validity-gated.

BM25 keyword scoring and semantic cosine similarity are blended, adjusted by recency decay, then passed through the validity gate so superseded versions never reach the model.

Result: 0 stale facts in top-5 recalls vs 4/4 for an accumulate-everything store, with 100% supersession accuracy. See the eval →

Supersession pipeline

Revised facts close cleanly.

Stage 1 — Metadata overlap

Candidate and existing facts are matched on structured keys (ticker, metric, subject) to find what could conflict.

Stage 2 — Deterministic decision

A rule engine classifies the relationship: SUPERSEDES, REFINES (a narrowing that enriches the old fact), CONFIRMS, or ADDS — no model required, fully reproducible.

Stage 3 — Optional LLM adjudication

For paraphrase-level conflicts, an optional Claude Haiku stage resolves ambiguity (SUPERSESSION_LLM_STAGE).

Result

The superseded version's validity window closes; it survives for audit but is excluded from present recall.

Human review — Memory GovernorNew

Low-confidence decisions queue at GET /v1/supersessions/review and in the console's Governance view. Confirm or reject with one call — a reject restores the old fact's validity, audited either way.

Tamper-evident audit chain

Proof built into every write.

Each event is appended to a SHA-256 hash chain — every row commits to the hash of the previous one, satisfying SEC 17a-4 / FINRA 4511 append-only requirements. Anyone can verify the chain: GET /v1/admin/audit/verify returns {"chain_valid": true} or detects where a record was altered. Export the full log (/v1/admin/audit/export) or reconstruct state at any past timestamp (/v1/snapshot?as_of=…). WORM mode object-locks the audit store for 17a-4 retention.

Flagship · Memory admission control

Govern what is allowed into memory.

Most memory layers write whatever the agent hands them. Lians puts a policy gate in front of the store — the differentiator no competitor offers.

Sensitive-data detection

PII, PHI, and MNPI are classified on the way in, so regulated content is handled by policy rather than by accident.

Source-trust validation

Writes are scored against source trust so low-trust inputs don't silently become durable facts.

Prompt-injection quarantine

Suspicious instructions are held out of recall until reviewed, instead of poisoning future context.

Vagueness pre-filterNew

Too-vague candidates are tagged and rejected before they can dilute recall — harvested from the Memory Governor vocabulary.

Review queue + modes

High-risk writes route to a review queue (/v1/admissions and the console's Governance view). Run admission in off, monitor, or enforce mode (ADMISSION_MODE).
