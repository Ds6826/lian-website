Memory that knows when it changed.

A memory control plane for AI agents working on time-sensitive, audited, and confidential data. Recall what was valid then — not just what looks similar now.

$ pip install lians-sdk[local]

mem.add(agent_id="desk",
        content="NVDA guidance raised to $40B",
        event_time="2025-11-19")

mem.recall_at(query="NVDA guidance", as_of="2025-03-01")
# → "$32B" — valid then, not today's $40B
$ npm install @lians-ai/lians

import { LiansClient } from "@lians-ai/lians"
const client = new LiansClient({ baseUrl, apiKey })

await client.recall({ agent_id: "desk",
                      query: "NVDA guidance" })
$ curl -X POST $LIANS/v1/recall \
   -d '{"agent_id":"desk","query":"NVDA guidance","as_of":"2025-03-01"}'

# → {"value":"$32B","valid_on":"2025-03-01"}

Capabilities

A control plane, not just a store.

Bitemporal recallpoint-in-time

Every fact carries event time and system time. Ask what an agent knew on any past date.

 Tamper-evident audit chainSEC 17a-4 / WORM

Each event enters an append-only SHA-256 hash chain. Verify the record, don't just trust it.

 Crypto-shred erasureGDPR Art. 17

Destroy a per-subject key to provably erase content while the audit trail survives.

 Memory admission controlFlagship

A policy gate on every write — PII/PHI/MNPI detection, source trust, prompt-injection quarantine. No competitor ships this.

 Relationship graphtemporal edges

Conflict-of-interest, related-party, and care-network queries — with no separate graph database.

 Memory Governorchange control

Memory updates land as proposed, versioned, audited, reversible changes — not silent overwrites.
