Self-improving memory with change control.

Memory Governor lets AI agents remember, improve, and recall safely by proposing, validating, versioning, auditing, and explaining every memory change — instead of silently overwriting facts.

The risk

Remembering more can mean trusting less.

Generic memory systems absorb facts as they arrive. That leaves agents exposed to stale facts, poisoned instructions, contradictions, overgeneralized preferences, and updates nobody can explain later. In serious environments, memory is not a cache — it is a decision surface.

The Lians approach

Every update becomes a governed proposal.

conversation / document / tool output
   → candidate memories
   → governor decisions
   → policy gates
   → audit commit
   → explainable recall

Lians compares candidates against existing memory, then decides whether to add, confirm, refine, supersede, contradict, quarantine, or ignore — before changing the durable record.

Vagueness admission filterNew

Governed by design

Memory with a control plane.

Memory proposals

Learned facts start as proposals — like pull requests for memory, not silent mutations.

Temporal recall

Ask what an agent knew at a point in time, not only what it knows now.

Supersession

Close stale validity windows so revised facts don't pollute current context.

Audit chain

Commit every change through an append-only, tamper-evident event trail.

Evidence links

Preserve the source and rationale behind material memory changes.

Policy gates

Apply rules before durable memory changes are allowed to commit.

Quarantine

Keep suspicious, conflicting, or low-confidence facts out of recall until reviewed.

Explainable recall

Show why a fact was returned, what it superseded, and what was suppressed.

The review API

Confirm or reject, with an audit trail.

# What is waiting for a human decision?
GET /v1/supersessions/review
# → flagged events: relation (SUPERSEDES | REFINES | …), confidence, rationale

# The engine got it right — accept the supersession.
PATCH /v1/supersessions/{memory_id}
  {"action": "confirm", "reviewer_note": "guidance revision verified"}

# The engine got it wrong — restore the old fact as currently valid.
PATCH /v1/supersessions/{memory_id}
  {"action": "reject"}

# Writes held by admission control (PII / PHI / MNPI / injection / vagueness):
GET  /v1/admissions?status=pending
POST /v1/admissions/{id}/resolve   {"action": "approve" | "reject"}
